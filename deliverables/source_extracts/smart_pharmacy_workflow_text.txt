Smart Pharmacy System Documentation
1. System Overview
Smart Pharmacy is a pharmacy management system designed around a fast POS workflow, strong stock
control, branch-level accountability, alerts, approvals, and owner visibility.
The system is built to support day-to-day pharmacy operations such as selling medicines, purchasing stock,
managing batches, tracking expiry, handling returns, recording expenses, closing cashier days, moving
stock between branches, and notifying responsible users when action is required.
The main principle is simple:
Every important business movement should be recorded, controlled by permissions, visible in
reports, and traceable through activity logs or notifications.
2. Main Roles
2.1 Owner
The Owner has full business control. This role can view and manage all branches, users, stock, sales,
expenses, reports, settings, alerts, approvals, and audit records.
Typical Owner responsibilities:
View dashboard performance.
Review sales, profit, expenses, and stock value.
Approve sensitive operations.
Verify or reject daily closings.
Review pending notifications.
Monitor stock alerts and expiry risks.
Manage system settings and users.
View reports across all branches.
2.2 Admin
Admin is almost the same as Owner in system control. This role is used for a trusted system administrator
or senior operator .
Typical Admin responsibilities:
Configure system setup.
Manage users and permissions.

2.3 Pharmacist
The Pharmacist handles pharmacy operations and medicine-related work. This role can sell, view stock,
manage prescriptions, create sales return requests, create stock adjustment requests, receive stock
transfers, record expenses, and submit daily closing.
Pharmacist cannot approve sensitive business actions such as final stock adjustment approval, sales return
approval, or daily closing verification.
Typical Pharmacist responsibilities:
Use POS.
View product and stock information.
Manage prescriptions.
Create sales return requests.
Create stock adjustment requests.
Receive stock transfers if assigned to destination branch.
Record expenses.
Submit daily closing.
2.4 Cashier
Cashier is focused on POS operations. This role sells products, prints receipts, records POS expenses,
creates sales return requests, and submits daily closings.
Cashier cannot approve returns, stock transfers, stock adjustments, or verify daily closing.
Typical Cashier responsibilities:
Search and sell products through POS.
Print receipts.
View today’s sales.
Record daily expenses.
Submit daily closing.
Create sales return request when customer brings items back.
2.5 Storekeeper
Storekeeper controls stock movement and inventory records. This role can manage products, suppliers,
purchases, stock adjustments, and stock transfers.
Storekeeper can create stock transfers and dispatch/receive them, but final approval remains Owner/
Admin.

The complete stock and money flow works like this:
Setup
→ Products + Units + Prices
→ Suppliers
→ Purchases
→ Inventory batches
→ POS Sales
→ Sales Returns if needed
→ Expenses
→ Daily Closing
→ Reports
→ Dashboard
→ Alerts + Notifications
The complete inventory flow works like this:
Purchase received
→ Inventory increases
→ POS sale
→ Inventory decreases
→ Sales return
→ Inventory may be restored if item is sellable
→ Stock adjustment
→ Inventory corrected, damaged stock removed, expired stock written off
→ Stock transfer
→ Source branch stock decreases, destination branch stock increases
→ Alerts monitor low stock and expiry
